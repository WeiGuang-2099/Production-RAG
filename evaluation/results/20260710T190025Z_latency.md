## Cache-hit latency (semantic cache, Redis backend)

With the stack up (`docker-compose up -d`), `CACHE_ENABLED=true`, and
`REDIS_URL=redis://localhost:6379/0`, the same question sent to `/chat`
twice: the first (uncached) call took ~8.5 s end to end (retrieval +
gpt-4o generation); repeat calls served from Redis took **~0.22 s**
wall-clock (three repeats: 0.220 / 0.219 / 0.230 s), which includes the
HTTP round-trip and output guardrails. The entry was verified to live in
Redis (`rag:cache:entry:...` + FIFO order list), not the in-memory
fallback. Note: the JSON `latency_ms` field of a cached response echoes
the original run's value by design (the stored result is returned as-is);
the wall-clock timing above is the honest hit latency.

## What moved and why

- Full-pipeline p50 dropped roughly 2x (+rerank / +graph stages, both
  corpora: ~1.4s -> ~0.64s) and p95 up to 2.5x (base6 +graph:
  2445 -> 989 ms).
- BM25 leg: index unpickle (grows with corpus) now happens once per
  process, not per query. Before, the +bm25 stage cost more on scale30
  (p50 1080 vs 1005 on base6 — a 10.7 MB pickle vs 2.4 MB); after, the
  stage adds ~0 over baseline on both corpora (311/326 ms vs 320/313 ms).
- Vector leg: the Qdrant connection (and its embedded store handle) is
  reused across queries instead of being rebuilt per query — the largest
  single win, visible in the baseline stage (~1010 -> ~315 ms) where no
  other component runs.
- +bm25/+graph stages: local legs now run inside the vector leg's shadow
  (parallel), so their incremental latency cost approaches zero.
- Quality metrics are unchanged by construction (deterministic RRF over
  identical inputs); verified per stage on both corpora — recall@5 / MRR /
  hit@5 are bit-identical between the before and after runs.

## Honest limits

- Single run per side; treat small deltas (<10%) as noise.
- The embedding API call is the irreducible floor of every stage (~300 ms
  here); further retrieval-side wins would have to come from embedding
  caching or a local embedder, not from store plumbing.
- The rerank stages still pay the Cohere round-trip (~320 ms p50 on top of
  baseline); that is the price of the MRR gain documented in the scale
  ablation.
